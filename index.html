Eclipse Debugger Step Filtering

Wednesday, April 4, 2012 5:11 PM Eastern

Spring proxies calls to methods on other objects. The proxying makes it difficult to step into the method. Previously, I was jumping to the defintion of the method I wanted to step into, setting a breakpoint, continuing execution, and removing the breakpoint. It worked, but was annoying.

Today, I found Eclipse's Step Filtering at Preferences > Java > Debug > Step Filtering. This will skip packages or classes when debugging. I am now filtering the following packages: com.mchange.v2.*, com.mysql.*, com.postresql.*, java.*, net.sf.cflib.*, org.apache.*, org.hibernate.*, org.postresql.*, org.slf4j.*, org.springframework.*, sun.*. I left the default java.lang.ClassLoader class filtered as well.

As I debug other areas of my applications, I expect I will tweak these filters. I will try to post updates here.

Git Log Branch Graphs

Tuesday, April 3, 2012 3:03 PM Eastern

There are two good ways to visualize branches: on the command line git log --graph --oneline --all --decorate --abbrev-commit and using gitk -all. There are lots of variants to these commands; use gitk branch1 branch2 to visual the two branches.

Bash Startup Files (profile, .bash_profile, .bash_login, .profile)

Wednesday, March 21, 2012 3:43 PM Eastern

When I first setup my work mac 10.7, I had some bash configuration options I wanted to set. The ~/.profile file existed, but no other bash startup files did. I read a little bit about the different files, didn't find any convincing arguments for me to care much, and put my configuration options in ~/.profile. Recently, I noticed that my tab completions and environment variables weren't getting set properly. I investigated a bit more today, and found the ~/.profile wasn't getting executed. I had a ~/.bash_login file! Bashrc and others taught me about the loading order. I had installed RVM recently, which created my ~/.bash_login file and caused my ~/.profile to be not executed.

Excluding Git Tracked Files

Thursday, March 15, 2012 5:33 PM Eastern

I have a file (.classpath) that git was tracking, but I didn’t want to see changes to with git status or commit them. The .gitignore file ignores untracked files, but not tracked ones. Some git index abuse allowed me to get what I wanted: git update-index --assume-unchanged <file>.
